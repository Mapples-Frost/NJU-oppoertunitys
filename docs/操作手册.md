# NJU Opportunity Radar 操作手册

这份手册默认你是第一次接触 GitHub Actions、Secrets 和命令行。照着做就行，看到不懂的词先不用慌。

项目仓库地址：

<https://github.com/Mapples-Frost/NJU-oppoertunitys>

## 1. 你最终会得到什么

配置好以后，系统会每天自动做这些事：

1. 从公开网页、竞赛平台、科研网站、微信公众号公开文章和手动入口抓取机会。
2. 自动去重、分类、打分、识别截止时间。
3. 生成一份邮件日报。
4. 如果邮箱配置正确，就把日报发到你的邮箱。
5. 把运行结果保存到仓库里的 `data/` 和 `logs/`。

现在代码已经在仓库里了。你主要需要做的是：配置邮箱密钥，然后在 GitHub 上点一次手动运行测试。

## 2. 先准备邮箱信息

你需要准备 6 个值：

```text
SMTP_HOST
SMTP_PORT
SMTP_USER
SMTP_PASS
MAIL_FROM
MAIL_TO
```

它们分别是什么意思：

| 名称 | 填什么 |
|---|---|
| `SMTP_HOST` | 邮箱服务商的 SMTP 服务器地址 |
| `SMTP_PORT` | SMTP 端口，常见是 `465` 或 `587` |
| `SMTP_USER` | 发件邮箱账号 |
| `SMTP_PASS` | 邮箱 SMTP 授权码或应用专用密码，不是普通登录密码 |
| `MAIL_FROM` | 发件人邮箱，一般和 `SMTP_USER` 一样 |
| `MAIL_TO` | 收件人邮箱，也就是你希望收到日报的邮箱 |

常见邮箱参考：

| 邮箱 | `SMTP_HOST` | `SMTP_PORT` | `SMTP_PASS` 通常是什么 |
|---|---|---|---|
| QQ 邮箱 | `smtp.qq.com` | `465` | QQ 邮箱设置里生成的 SMTP 授权码 |
| 163 邮箱 | `smtp.163.com` | `465` | 163 邮箱设置里生成的授权码 |
| Gmail | `smtp.gmail.com` | `465` 或 `587` | Google 账号的应用专用密码 |
| Outlook | `smtp.office365.com` | `587` | 账号密码或应用密码，取决于账号安全设置 |

重要提醒：

- 不要把授权码发给别人。
- 不要把授权码写进 `README.md`、`config/email.yml` 或任何仓库文件。
- GitHub Secrets 只能看到名字，看不到保存后的值。如果填错了，直接删掉或覆盖重填。

## 3. 在 GitHub 配置 Secrets

打开仓库：

<https://github.com/Mapples-Frost/NJU-oppoertunitys>

按下面步骤点：

1. 进入仓库首页。
2. 点上方的 `Settings`。
3. 在左侧找到 `Security` 区域。
4. 点 `Secrets and variables`。
5. 点 `Actions`。
6. 确认在 `Secrets` 标签页。
7. 点 `New repository secret`。
8. 在 `Name` 里填密钥名，例如 `SMTP_HOST`。
9. 在 `Secret` 里填对应的值，例如 `smtp.qq.com`。
10. 点 `Add secret`。
11. 重复 6 次，把下面 6 个都添加进去：

```text
SMTP_HOST
SMTP_PORT
SMTP_USER
SMTP_PASS
MAIL_FROM
MAIL_TO
```

如果你用 QQ 邮箱，一个例子长这样：

```text
SMTP_HOST = smtp.qq.com
SMTP_PORT = 465
SMTP_USER = 你的QQ邮箱@qq.com
SMTP_PASS = 你的SMTP授权码
MAIL_FROM = 你的QQ邮箱@qq.com
MAIL_TO = 你想接收日报的邮箱
```

注意：`SMTP_PASS` 要填授权码，不要填 QQ 登录密码。

GitHub 官方路径参考：仓库 `Settings` -> `Secrets and variables` -> `Actions` -> `New repository secret`。

## 4. 第一次手动运行

Secrets 配好后，先不要等第二天，手动运行一次测试。

操作步骤：

1. 打开仓库首页。
2. 点上方的 `Actions`。
3. 左侧点 `Daily Opportunity Radar`。
4. 右侧点 `Run workflow`。
5. Branch 选择 `main`。
6. 再点绿色的 `Run workflow`。
7. 等几分钟，页面会出现一条新的运行记录。

看结果：

- 绿色对勾：运行成功。
- 红色叉号：运行失败，点进去看失败日志。
- 黄色转圈：还在运行。

如果 SMTP 配好了，运行成功后你应该能收到一封标题类似下面的邮件：

```text
[NJU Opportunity Radar] 2026-05-24: 发现 50 个新机会，0 个高优先级
```

如果没有收到邮件，但 Actions 是绿色的，优先检查这几件事：

1. `SMTP_PASS` 是授权码，不是登录密码。
2. `MAIL_TO` 没写错。
3. 邮件没有进入垃圾箱。
4. 邮箱服务商是否开启了 SMTP 服务。

## 5. 每天自动运行

仓库里的 workflow 文件是：

```text
.github/workflows/daily.yml
```

它已经配置为每天运行一次：

```yaml
cron: "0 0 * * *"
```

GitHub Actions 的默认 cron 使用 UTC 时间。`0 0 * * *` 对应北京时间 / 上海时间每天 08:00。

需要知道两点：

- GitHub 定时任务不保证精确到分钟，偶尔可能延迟。
- 公共仓库如果长期没有活动，GitHub 可能暂停定时任务。偶尔提交一次或手动运行一次即可恢复活跃。

## 6. 看运行结果

每次运行后，系统会更新这些文件：

```text
logs/latest_run.json
logs/latest_email.txt
logs/latest_email.html
data/opportunities.sqlite
```

你主要看这两个就够了：

| 文件 | 用途 |
|---|---|
| `logs/latest_email.txt` | 最近一次日报的纯文本版 |
| `logs/latest_run.json` | 最近一次运行状态，包括成功源、失败源、新机会数量 |

如果邮件没收到，可以先打开 `logs/latest_email.txt` 看日报有没有生成。

## 7. 手动添加微信群或公众号里的机会

有些机会来自微信群、QQ群、公众号。系统不会强爬这些私域来源，但支持你主动转发或粘贴公开文章链接。你可以手动粘贴到：

```text
inbox/manual.md
```

格式如下：

```markdown
## 华为某某挑战赛
链接：https://example.com
来源：微信群转发
内容：这是一个面向 AI 工程和算法方向的挑战赛，适合组队参加。
截止：2026-06-15
```

注意：

- 每条机会必须用 `##` 开头。
- `链接`、`来源`、`内容`、`截止` 后面用中文冒号或英文冒号都可以。
- 填完后提交到 GitHub，下一次运行时它会和其他机会一起进入日报。

最简单的 GitHub 网页操作：

1. 打开仓库里的 `inbox/manual.md`。
2. 点右上角铅笔图标编辑。
3. 粘贴新机会。
4. 页面底部点 `Commit changes`。

V2 也支持更简单的公众号文章链接池。打开：

```text
inbox/wechat_articles.md
```

把公开微信公众号文章链接直接粘进去即可，例如：

```text
https://mp.weixin.qq.com/s/xxxxx
```

系统会自动抽取文章标题、公众号名、正文，并和其他竞赛/科研网站一起分类打分。

如果你想把公众号文章“转发到邮箱”再让系统读，可以额外配置 IMAP Secrets：

```text
IMAP_HOST
IMAP_PORT
IMAP_USER
IMAP_PASS
```

建议使用专用邮箱或专用文件夹，不要让系统读取你的私人主邮箱。

## 8. 本地试运行

如果你想在电脑上试跑：

1. 打开 PowerShell。
2. 进入项目目录：

```powershell
cd D:\NJU-oppertunity
```

3. 安装依赖：

```powershell
python -m pip install -r requirements.txt
```

4. 只测试抓取和日报渲染，不写数据库、不发邮件：

```powershell
python -m radar.main --config config --dry-run
```

5. 正常运行，写入本地数据库和日志，但不发邮件：

```powershell
python -m radar.main --config config
```

6. 本地发送邮件测试。先临时设置环境变量：

```powershell
$env:SMTP_HOST="smtp.qq.com"
$env:SMTP_PORT="465"
$env:SMTP_USER="你的邮箱"
$env:SMTP_PASS="你的授权码"
$env:MAIL_FROM="你的邮箱"
$env:MAIL_TO="收件邮箱"
python -m radar.main --config config --send-email
```

PowerShell 窗口关掉后，这些临时环境变量就没了。

## 9. 常见问题

### 9.1 Actions 成功了，但没有邮件

按顺序检查：

1. GitHub Secrets 里 6 个值是否都存在。
2. `SMTP_PASS` 是否是授权码。
3. 发件邮箱是否开启 SMTP。
4. `MAIL_TO` 是否写错。
5. 垃圾箱里有没有。
6. 点进 Actions 日志，看 `Run radar` 步骤有没有 SMTP 报错。

### 9.2 Actions 失败，提示 pip install 失败

可能是网络或 PyPI 临时波动。可以点失败记录右上角的 `Re-run jobs` 重试。

### 9.3 日报里有很多“系统异常”

这是正常的第一版现象。原因可能是：

- 目标网站用了动态渲染。
- 目标网站拒绝爬虫访问。
- 某些南大站点 HTTPS 握手失败。
- 网站结构变了，选择器没有匹配到列表。

系统会继续抓可访问的源，并把失败源列在日报底部。

V2 日报还会显示“源覆盖状态”，例如 `competition_pack`、`research_pack`、`wechat_pack` 各自成功了多少源、发现了多少候选机会。这能帮助你判断今天是竞赛源、科研源还是公众号源出了问题。

### 9.4 想新增一个网站

优先改：

```text
config/sources.yml
```

复制一个现有 `html_list` 源，然后改这些字段：

```yaml
id: unique_source_id
name: 网站名称
list_url: "列表页地址"
base_url: "网站根地址"
tags: [AI, 竞赛]
category_hint: AI竞赛
```

如果你不确定选择器，先用最简单的：

```yaml
list_selector: "a"
link_selector: "a"
detail_required: false
```

能跑起来以后，再慢慢调精。

### 9.5 想改每天发送时间

编辑：

```text
.github/workflows/daily.yml
```

当前是：

```yaml
cron: "0 0 * * *"
```

因为 GitHub 默认按 UTC 算，所以：

| 中国时间 | cron |
|---|---|
| 08:00 | `0 0 * * *` |
| 09:00 | `0 1 * * *` |
| 20:00 | `0 12 * * *` |

改完后提交到 `main` 分支。

- [ ] 手动点 `Run workflow` 后能跑成功。
- [ ] `logs/latest_email.txt` 有日报内容。
